# 周常任务求解器技术文档

## 概述

周常任务求解器为 MaaFgo 提供自动计算最优刷本方案的能力。给定当前周的御主任务（Master Mission）条件，求解器通过整数线性规划（ILP）计算出消耗最少 AP 的刷本组合。

目标服务器为国服（B服）。国服周常任务时间表确定（跟随日服历史），所有数据预下载到本地，运行时完全离线。

## 核心算法

### 问题建模

周常任务求解本质上是一个**整数线性规划问题**：

```
最小化:  Σ(cost_i × x_i)      — 总 AP 消耗
约束:    Σ(A_ji × x_i) >= b_j — 每个任务的目标完成数 >= 要求数
         x_i >= 0, x_i ∈ Z    — 各副本刷本次数为非负整数
```

其中：
- **x_i**: 第 i 个 Free 副本的刷本次数（求解目标）
- **A_ji**: 副本 i 对任务 j 的贡献值（如每次击杀多少个目标职阶敌人）
- **b_j**: 任务 j 的目标数（如"击败15个 Saber 职阶敌人"中的 15）
- **cost_i**: 副本 i 的 AP 消耗

### 求解引擎

使用 [HiGHS](https://highs.dev/) 求解器（通过 `highspy` Python 包，~3MB）。HiGHS 是 scipy 内部使用的同一求解器，支持线性规划（LP）和整数线性规划（ILP）。

### 与 Chaldea 的关系

本求解器的算法逻辑移植自 Chaldea 的 `MissionSolver`。Chaldea 使用 JavaScript 版 GLPK（GNU Linear Programming Kit），我们使用 Python 版 HiGHS，两者都是成熟的开源 LP/ILP 求解器，对同一问题产生相同的最优解。

## 数据策略

### 全离线方案

所有数据预下载到 `agent/mission_solver/`，用户运行时零网络请求。

| 数据 | 本地文件 | 说明 |
|---|---|---|
| Free 副本敌人数据 | `quest_enemies_CN.json` | 所有主线 Free 副本的敌人信息 |
| 主线 War/副本列表 | `wars_CN.json` | 主线章节及其包含的 Free 副本列表 |
| 周常任务时间表 | `master_missions_CN.json` | 国服历史周常任务，按周索引 |

### 数据来源

数据来自 [Atlas Academy API](https://api.atlasacademy.io)：

| API 端点 | 用途 |
|---|---|
| `GET /export/{region}/nice_master_mission.json` | 周常任务列表及条件 |
| `GET /nice/{region}/quest/{questId}/{phase}` | 副本的敌人信息（职阶、特性等） |
| `GET /nice/{region}/war/{warId}` | 主线章节的副本列表 |

### 数据更新

开发者运行 `tools/update_quest_data.py` 下载最新数据并提交到 git，普通用户无需操作。

需要更新的时机：
- 国服开放新主线章节（新增 Free 副本）
- 补充新的周常任务时间表

### matcher.py — 条件匹配

核心函数 `count_mission_target(mission, quest) -> int`，计算一次刷副本对任务的贡献值。

#### 支持的条件类型

| 条件类型 | 说明 | 匹配逻辑 |
|---|---|---|
| `trait` | 按敌人特性 | 遍历敌人，检查 traits 是否包含 target_ids |
| `enemyClass` | 按敌人职阶 | 遍历敌人，匹配 class_id |
| `servantClass` | 按从者职阶 | 遍历敌人，要求 is_servant=true 且匹配 class_id |
| `enemyNotServantClass` | 按非从者职阶 | 遍历敌人，要求 is_servant=false 且匹配 class_id |
| `enemy` | 按具体敌人 | 遍历敌人，匹配 svt_id |
| `quest` | 按副本 | 匹配 quest.id |
| `questTrait` | 按副本特性 | 匹配 quest.individuality |

#### 条件组合逻辑

- 单条件内多个 target_ids：`use_and=true` 时要求全部匹配，`false` 时任一匹配
- 多条件间：`cond_and=true` 时要求所有条件满足，`false` 时任一满足

### solver.py — ILP 求解

```python
import highspy

def solve(quests: list[QuestPhase], missions: list[Mission]) -> SolveResult:
    """
    求解最优刷本方案

    输入:
      quests   — 候选 Free 副本列表
      missions — 当前周常任务列表

    输出:
      SolveResult:
        plan     — {quest_id: 刷本次数}
        total_ap — 总 AP 消耗
        total_runs — 总刷本次数
        details  — 每个副本对每个任务的贡献明细
    """
```

求解流程：

```
1. 构建贡献矩阵 A[m][n]
   m = 任务数, n = 副本数
   A[j][i] = count_mission_target(missions[j], quests[i])

2. 过滤：移除对所有任务贡献为 0 的副本（减小问题规模）

3. 构建 ILP 问题：
   - n 个整数变量 x_i >= 0（各副本刷本次数）
   - m 个约束 Σ(A[j][i] * x_i) >= b[j]
   - 目标函数 min Σ(cost_i * x_i)

4. 调用 HiGHS 求解

5. 格式化输出
```

## 命令行使用

```bash
# 自动匹配当前周的国服周常任务并求解
python -m agent.mission_solver

# 指定主线进度（只使用已开放的 Free 副本）
python -m agent.mission_solver --progress 308

# 输出示例
# === 当前周常任务 (2026-04-14 ~ 2026-04-21) ===
#  1. 击败 15 个「Saber」职阶的敌人
#  2. 击败 15 个「拥有[神性]特性」的敌人
#  3. 完成 3 次「拥有[炎]特性」的关卡
#  ...
#
# === 最优方案 (总 AP: 120, 总次数: 6) ===
#  巴比伦尼亚 - 密林 (Free)  ×3  (AP 63)  → Saber×9, 神性×6
#  迦勒底之门 - 弓阶修炼场   ×2  (AP 40)  → Archer×10
#  ...
```

## 后续扩展

### 集成到 MaaFgo pipeline

新增 custom_action `solve_weekly_missions`：
1. 求解器计算最优方案
2. 自动调度刷对应副本的 pipeline 任务
3. 执行完成后标记任务完成

### 日服支持

`data_loader.py` 接口已预留 `region` 参数，扩展日服只需：
1. 添加 `fetch_current_missions("JP")` 在线获取函数（urllib.request）
2. 下载日服副本敌人数据 `quest_enemies_JP.json`

### 活动任务支持

扩展 `data_loader.py` 加载活动 War 的副本数据，核心算法无需改动。
